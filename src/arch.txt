Классы
Task, Epic, Subtask, ManagerTasks, Enum со статусами задачи.
Epic и Subtask будут наследоваться от Task.
Epic будет содержать список Subtask.

Задачи будут храниться HashMap, ключом будет id, значение сама задача.
Хранение реализовать в ManagerTasks, 3 разных Map для разных задач.
Создать методы добавления.

ManagerTasks должен реализовать:
     Получение списка всех задач.
     Удаление всех задач.
     Получение по идентификатору.
     Создание. Сам объект должен передаваться в качестве параметра.
     Обновление. Новая версия объекта с верным идентификатором передаётся в виде параметра.
     Удаление по идентификатору.
 **Дополнительные методы:**
     Получение списка всех подзадач определённого эпика.
